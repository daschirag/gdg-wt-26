# AQE — Storage-Aware Approximate Query Engine

> LSM-Tree based columnar analytical engine with sampling-driven approximate query processing.  
> Written in Rust (edition 2024) with Python tooling for data generation and benchmarking.

### Data Flow

1. **Ingestion** — CSV data is loaded via `csv_loader.rs` or generated by `datagen.rs` / `tools/gen_data.py`
2. **Flush** — Rows are written to row-based `.aqe` SSTables via `sstable/writer.rs`
3. **Compaction** — Background thread merges `.aqe` files into directory-based columnar segments via `compaction/merger.rs` → `columnar/writer.rs`
4. **Query** — SQL is parsed, planned (FastPath / Columnar / Row / Mixed), and executed with optional sampling for approximate results
5. **Result Scaling** — Sampled results are scaled by `Estimator::scale_result()` to estimate full-dataset values

## Configuration

### `config.toml`

| Parameter | Type | Default | Description |
|---|---|---|---|
| `accuracy_target` | f64 | 0.70 | Target accuracy for approximate queries (0.0–1.0) |
| `k` | f64 | 2.0 | Exponent for sampling rate calculation: `rate = accuracy^k` |
| `seed` | u64 | 42 | Deterministic RNG seed for reproducible sampling |
| `bloom_fpr` | f64 | 0.01 | Bloom filter false-positive rate |
| `memtable_row_limit` | u64 | 1,048,576 | Max rows before memtable flush |
| `memtable_size_limit` | u64 | 104,857,600 | Max bytes before memtable flush (~100 MB) |
| `min_group_rows` | u64 | 50 | Minimum rows for a GROUP BY bucket to be reported |
| `low_confidence_threshold` | u64 | 100 | Below this count, confidence is flagged `Low` |
| `verify_crc` | bool | false | Enable per-row CRC32 verification on read |
| `compaction_strategy` | String | "stcs" | `"stcs"` (size-tiered) or `"lcs"` (leveled) |
| `compaction_interval_secs` | u64 | 30 | Background compaction check interval |
| `min_compaction_files` | u64 | 4 | Minimum .aqe files in a size bucket to trigger STCS |
| `row_group_size` | u64 | 1024 | Row group size for columnar scan chunking |
| `lcs_l1_max_bytes` | u64 | 10,485,760 | LCS L1 size threshold (~10 MB) |
| `lcs_l2_max_bytes` | u64 | 104,857,600 | LCS L2 size threshold (~100 MB) |
| `max_segment_rows` | u64 | 1,000,000 | Max rows per columnar segment (triggers splitting) |

### `schema.toml`

Defines the table schema. Currently 5 columns:

| Column | Type | Description |
|---|---|---|
| `user_id` | u64 | Sequential user identifier (primary key for dedup) |
| `status` | u8 | 0=active (70%), 1=inactive (20%), 2=banned (10%) |
| `country` | u8 | Country code 0–4 (uniform distribution) |
| `timestamp` | u64 | Unix millisecond timestamp (2020–2024 range) |
| `level` | i64 | User level 0–1000 (uniform distribution) |

## Storage Layer

### Memtable

**File:** `src/storage/memtable.rs`

In-memory write buffer backed by a `BTreeMap<u64, RowDisk>` keyed on `user_id`.

| Method | Description |
|---|---|
| `new()` | Creates empty memtable |
| `insert(row)` | Assigns global sequence number via `AtomicU64`, inserts by user_id |
| `get(user_id)` | Point lookup by key |
| `clear()` | Drops all entries |
| `row_count()` | Returns `data.len()` |

Uses a global `AtomicU64` counter (`GLOBAL_SEQ`) for LSM sequence ordering.

---

### SSTable (Row Format)

**Files:** `src/storage/sstable/writer.rs`, `src/storage/sstable/reader.rs`

Binary format for row-oriented storage (`.aqe` files). These are the **L0 files** consumed by compaction.

#### File Layout

```
┌─────────────┬──────────────┬──────────────────┬────────────┬──────────┐
│ Magic (4B)  │ Meta Len (4B)│ Metadata (bincode)│ Bloom Filter│ Rows...  │
│ "SQTE"      │ u32 LE       │ SSTableMetadata   │ (bincode)  │ (bincode)│
└─────────────┴──────────────┴──────────────────┴────────────┴──────────┘
```

#### `SSTableWriter`

| Method | Description |
|---|---|
| `write_sstable_to_file(path, rows, config)` | Convenience: creates writer, writes with 1% FPR |
| `write_sstable(rows, fpr, generate_crc)` | Writes magic → metadata (with CRC32 checksum) → bloom filter → rows |

Column metadata uses `col_N` naming (index-based, not schema names).

#### `SSTableReader`

| Method | Description |
|---|---|
| `read_rows_profiled(verify_crc)` | Full read with timing profile. Verifies magic + metadata checksum |
| `read_rows()` | Shorthand with CRC verification enabled |
| `get_metadata()` | Reads only the metadata header (for fast-path queries) |
| `get_bloom_filter()` | Reads only the bloom filter (for pre-filtering) |

### Columnar Segments

**Files:** `src/storage/columnar/writer.rs`, `src/storage/columnar/reader.rs`, `src/storage/columnar/segment.rs`

The **primary storage format** after compaction. Each segment is a directory containing individual `.col` files per column plus a `meta.toml` metadata file and a `bloom.bin` bloom filter.

#### Segment Directory Layout

```
data/segments/seg_000/
├── meta.toml       # SSTableMetadata serialized as TOML
├── bloom.bin       # Serialized BloomFilterWrapper (bincode)
├── user_id.col     # Column data (delta-encoded)
├── status.col      # Column data (RLE-encoded, low cardinality)
├── country.col     # Column data (RLE-encoded, low cardinality)
├── timestamp.col   # Column data (delta-encoded)
└── level.col       # Column data (bincode, AlignedVec<i64>)
```

#### `ColumnarWriter`

| Method | Description |
|---|---|
| `write_segment(path, rows, config, fpr)` | Entry point: transposes rows → columns, encodes, writes files |
| `write_segment_from_iter(path, iter, config, fpr)` | Iterator-based variant (still collects internally) |

**Encoding Selection Heuristic:**
1. **RLE** — If `distinct_count < row_count / 10` (low cardinality: status, country)
2. **Delta** — If column is `timestamp` or `user_id` (monotonic-like)
3. **Bincode** — Fallback (serialized via `AlignedVec` for SIMD-friendly layout)

#### `ColumnarReader`

| Method | Description |
|---|---|
| `new(path, seg_idx)` | Loads metadata from `metadata.json` (priority) or `meta.toml` |
| `read_column_i64(name)` | Reads and decodes a column → `AlignedVec<i64>` (dispatches on encoding) |
| `read_column_f64(name)` | Same for f64 columns |
| `read_column_bytes(name)` | Raw bytes for manual decoding |
| `read_column_runs_i64(name)` | Deserialize directly to `Vec<RleRun<i64>>` (no decode) |
| `open_column(name)` | Returns `ColumnFileHandle` for lazy/streamed access |
| `read_all_rows(config)` | Reconstructs `Vec<RowDisk>` from all columns (used by `resplit`) |

#### `ColumnFileHandle`

Provides lazy, cached access to column data with RLE run caching:

| Method | Description |
|---|---|
| `get_runs_i64()` | Cached deserialization to `Vec<RleRun<i64>>` |
| `get_runs_u8()` | Cached deserialization to `Vec<RleRun<u8>>` |
| `read_all_i64()` | Full decode to `AlignedVec<i64>` |
| `verify_crc()` | Validates column file CRC32 against metadata |

#### `Segment`

| Method | Description |
|---|---|
| `new(path, metadata)` | Direct construction |
| `load_from_dir(path)` | Loads metadata from `meta.toml` |

### Columnar Encoding

#### `src/storage/columnar/encoding/rle.rs`

Run-Length Encoding for low-cardinality columns (status, country).

| Struct | Description |
|---|---|
| `RleRun<T>` | `{value: T, length: u32}` — a single run |

| Method | Description |
|---|---|
| `encode(data)` | Compresses consecutive equal values into runs |
| `decode(runs)` | Expands runs back to flat Vec |
| `decode_range(runs, start, end)` | Positional slice decode |
| `decode_chunk_i64(runs, count, cursor)` | Stateful chunked decoding with cursor |
| `seek_to(runs, target)` | Binary search to a logical position → `(run_idx, offset)` |
| `aggregate_counts_i64(runs)` | COUNT GROUP BY directly on runs (no decode) |
| `aggregate_counts_i64_sampled(runs, rate)` | Sampled variant (scales run lengths) |
| `aggregate_sum_by_u8_i64(keys, vals)` | SUM grouped by u8 key (co-iterated RLE runs) |
| `aggregate_sum_by_u8_i64_sampled(keys, vals, rate)` | Sampled variant with alignment maintenance |

Key optimization: `aggregate_*` methods operate directly on RLE runs without full decode, achieving O(runs) instead of O(rows).

#### `src/storage/columnar/encoding/delta.rs`

Delta encoding for monotonic/sequential columns (user_id, timestamp).

| Struct | Description |
|---|---|
| `DeltaEncoded<T>` | `{base: T, deltas: Vec<i64>}` — stores differences |

| Method | Description |
|---|---|
| `encode_i64(values)` | Computes delta chain from first value |
| `decode_i64(encoded)` | Reconstructs values via prefix-sum |

Decoding produces `AlignedVec<i64>` for SIMD-friendly downstream processing.

### Bloom Filters

**File:** `src/storage/bloom/filter.rs`

Wraps the `bloomfilter` crate's `Bloom<u64>` with custom serde support (serializes bit vector + SIP keys for portable storage).

| Method | Description |
|---|---|
| `new(num_items, fpr)` | Creates filter sized for expected item count and false-positive rate |
| `insert(key)` | Adds a u64 key |
| `contains(key)` | Probabilistic membership check |

Used for `user_id` equality filter optimization (bloom → skip entire SSTable).

### Compaction Engine

#### `src/storage/compaction/engine.rs` — `CompactionEngine`

| Method | Description |
|---|---|
| `run_compaction(storage)` | Main entry: validates metadata → plans → executes merges → updates registry |
| `verify_storage_metadata(dir, config)` | Checks all segment `meta.toml` files match current schema |
| `next_segment_id(dir)` | Auto-increments segment ID based on existing `seg_NNN` directories |

#### `src/storage/compaction/scheduler.rs` — `CompactionScheduler`

| Method | Description |
|---|---|
| `start_background_thread(storage)` | Spawns daemon thread, runs compaction every `compaction_interval_secs` |
| `compact_now(storage)` | Manual one-shot compaction (with AtomicBool guard against concurrent runs) |

#### `src/storage/compaction/stcs.rs` — Size-Tiered Compaction Strategy

Groups `.aqe` files by size (2× ratio bucketing). Triggers when a bucket has ≥ `min_compaction_files` files.

#### `src/storage/compaction/lcs.rs` — Leveled Compaction Strategy

Simplified LCS: compacts all L0 `.aqe` files when count ≥ 4 **or** total size exceeds `lcs_l1_max_bytes`.

#### `src/storage/compaction/merger.rs` — `SegmentMerger`

K-way merge using `BinaryHeap`. Deduplicates by `user_id` (keeps latest by sequence number). Splits output into multiple segments if row count exceeds `max_segment_rows`.

Pipeline: Read `.aqe` → heap-merge → dedup → `ColumnarWriter::write_segment()` → cleanup source files.

## Query Layer

### SQL Parser

**File:** `src/query/parser.rs`

Whitespace-tokenized parser supporting a subset of SQL:

```sql
SELECT <aggregation> FROM <table> [WHERE <col> <op> <val>] [GROUP BY <col>]
```

**Supported aggregations:** `COUNT(*)`, `SUM(col)`, `AVG(col)`  
**Supported operators:** `=`, `>`, `<`, `>=`, `<=`

Produces a `QueryPlan` (defined in `ast.rs`):

```rust
pub struct QueryPlan {
    pub aggregation: Aggregation,  // Count | Sum(col) | Avg(col)
    pub table: String,
    pub filter: Option<Filter>,    // {column, op, value}
    pub group_by: Option<String>,
}
```

### Query Planner

**File:** `src/query/planner.rs`

Chooses an `ExecutionPath` based on available storage and query characteristics:

| Path | Condition | Description |
|---|---|---|
| `FastPath` | No filter, no GROUP BY, no `.aqe` files | Metadata-only: reads pre-computed sums/counts from segment `meta.toml` |
| `Columnar` | Segments exist, no `.aqe` files | Pure columnar scan with encoding-aware aggregation |
| `Row` | Only `.aqe` files exist | Row-oriented bincode deserialization |
| `Mixed` | Both segments and `.aqe` files | Runs both pipelines and merges results |

### Execution Pipelines

#### `src/query/executor/pipeline.rs` — Row Pipeline

Handles Row and Mixed paths. Has its own internal metadata fast-path.

**Execution stages:**
1. **Metadata Fast-Path** — If no filter/GROUP BY, aggregate from metadata only
2. **Sampling Rate** — Compute SST-level and row-level rates via `AccuracyCalculator`
3. **Bloom Filter** — Skip SSTables that definitely don't contain the filtered user_id
4. **SSTable Sampling** — Weighted random selection of SSTables
5. **Column Loading** — Reads needed columns from columnar reader (or falls back to row reader)
6. **Filtering** — Applies WHERE clause on loaded column data
7. **Row Sampling & Dedup** — Hash-based row selection + user_id deduplication
8. **Aggregation** — Delegates to `Aggregator`

#### `src/query/executor/col_pipeline.rs` — Columnar Pipeline

Optimized for pure columnar segments. Uses Rayon for parallel segment scanning.

**Scan Strategies:**
- `RunAggregate` — Aggregates directly on RLE runs (no filter path). Fastest.
- `LazyGroupDecode` — Decodes row groups lazily, applies filter, then aggregates (filter path).

**Key optimizations:**
- RLE-aware COUNT/SUM aggregation (`aggregate_counts_i64_sampled`, `aggregate_sum_by_u8_i64_sampled`)
- Row-group level sampling via `ColumnSampler::should_sample_row_group()`
- Parallel segment scanning via `rayon::par_iter()`
- Final result scaling via `Estimator::scale_result()`

> **Note:** There is an unused function `scaler_count_final_add()` at the bottom of this file. See [Deprecated & Unused Code](#deprecated--unused-code).

### Aggregator

**File:** `src/query/executor/aggregator.rs`

| Method | Description |
|---|---|
| `aggregate(rows)` | Row-oriented: dispatches to scalar or group aggregation |
| `aggregate_columnar(i64_cols, f64_cols, ...)` | Columnar: operates on `AlignedVec` column vectors |

Supports `Count`, `Sum`, `Avg` with optional GROUP BY. Group results include per-group `ConfidenceFlag`.

## AQP Layer (Approximate Query Processing)

### Accuracy Calculator

**File:** `src/aqp/accuracy.rs`

| Method | Description |
|---|---|
| `calculate_sampling_rate(accuracy, k)` | `accuracy^k` clamped to [0.0001, 1.0]. At defaults (0.70, 2.0) → 0.49 |
| `split_sampling_rate(rate)` | Splits into SST-level and row-level: both = `sqrt(rate)` |
| `calculate_required_sample_size(variance, accuracy, k)` | Adaptive sample size from variance (min 100) |

> **Note:** `calculate_required_sample_size` is **never called** anywhere. See [Deprecated & Unused Code](#deprecated--unused-code).

### Estimator

**File:** `src/aqp/estimator.rs`

Scales sampled results to full-dataset estimates:
- **COUNT / SUM** → `value * (1 / sampling_rate)`
- **AVG** → no scaling needed (sample mean ≈ population mean)

Works on both `Scalar` and `Groups` variants.

### Samplers

#### `src/aqp/sampler/sst_sampler.rs` — SSTable-Level Sampling

Weighted random selection using `WeightedIndex` (weighted by row count). Uses `ChaCha8` RNG for reproducibility. Selects `max(1, floor(rate × total))` distinct SSTables.

#### `src/aqp/sampler/row_sampler.rs` — Row-Level Sampling

Deterministic hash-based: `hash(seed, offset) % 1000 < rate*1000`. No RNG state, fully reproducible for same seed and position.

| Method | Description |
|---|---|
| `sample_rows(rows, rate, seed, ...)` | Filters a `Vec<RowDisk>` by hash-based sampling |
| `is_row_sampled(offset, rate, seed)` | Stateless check for a single row's inclusion |

> **Note:** `sample_rows()` is **never called** — only `is_row_sampled()` is used by the pipeline. See [Deprecated & Unused Code](#deprecated--unused-code).

#### `src/aqp/sampler/col_sampler.rs` — Column/Segment Sampling

| Method | Description |
|---|---|
| `sample_segments(segments, rate, seed)` | Shuffled segment selection (StdRng) |
| `should_sample_row_group(seg_idx, group_idx, rate, seed)` | Delegates to `RowSampler::is_row_sampled()` |

> **Note:** `sample_segments()` is **never called** — the columnar pipeline does its own inline segment sampling. See [Deprecated & Unused Code](#deprecated--unused-code).
